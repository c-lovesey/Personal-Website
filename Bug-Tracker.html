Bug Tracker

Software Development can be difficult as it is hard to see changes and what problems and changes occurred at different times during the development process, that is why programs like GitHub exist, it allows developers to view all changes on a piece of software.

This program aims to solve some of the issues that can arise when developing software, the software acts as a centralized area for developers to report bugs, this program allows the user to report how, when and why a bug occurred.

The program can utilize either a centralized database or a local CSV file to store reports, The user can add new applications, versions and environments to the database which can then be used to create reports.

The reports includes basic details about the bug such as the title, the appication it occurs in, the version and the environment.

The more advanced details are the confirmation status of a bug, this allows user requests to be submitted to the database and confirmed, the status of a report, either open or closed which shows if anyone is actively working on the bug. The resolution of the bug, the priority of the bug and if the bug is fixed or has a fixed version.

The main page of the application shows all the actions the user can perform on the left navigation bar. and the rest of the screen is taken up by the search interface and the viewer interface.

You can look at the code on my GitHub page, both the application and stored procedures are included.
